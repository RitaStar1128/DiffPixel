# DiffPixel Screenshot Plan

## Direction

Use a fictional web page instead of a real brand or unrelated website. The page should clearly feel like a design QA demo, so users understand that DiffPixel works on arbitrary pages and is not affiliated with the page shown in the screenshots.

Recommended fictional page name: `Pixel QA Demo`.

## What Strong Extension Listings Tend To Do

- Show the core value in the first screenshot.
- Use real product UI rather than abstract marketing art.
- Demonstrate a visible before/after or action/result relationship.
- Keep captions short and benefit-led.
- Include privacy or low-permission reassurance when it is a differentiator.

## Five Screenshot Set

All screenshots should be exported at `1280x800` for the Chrome Web Store. Create Japanese and English localized versions with the same layout.

### 1. Compare Reference and Page

Caption: `Compare designs directly on the page`

Show the `Pixel QA Demo` landing page with a semi-transparent reference image overlaid. The DiffPixel floating panel is open on the right. This should immediately communicate the product's single purpose.

### 2. Spot Differences Fast

Caption: `Use Difference mode to find visual drift`

Show the same page with Difference blend mode enabled. Highlight misaligned buttons, text blocks, or card spacing with visible color contrast. This is the strongest "why install it" screenshot.

### 3. Fine Tune Scale and Position

Caption: `Adjust scale, opacity, and position precisely`

Show the controls area with scale set to a decimal value such as `1.05`, plus X/Y and opacity controls. The overlay should visibly align more closely with the page.

### 4. Check Spacing With Grid

Caption: `Turn on a grid for spacing checks`

Show a card layout or form section with a clear grid overlay. The grid should be visible but not so heavy that it hides the page.

### 5. Review Multiple States

Caption: `Manage multiple reference layers locally`

Show multiple layers in the panel, such as `Desktop`, `Modal`, and `Mobile crop`, with visibility and lock controls. Add a small privacy reassurance in the screenshot: `Images stay on your device`.

## Visual Requirements

- Use a clean browser frame.
- Keep the fictional page neutral and relevant to UI review.
- Avoid real brands, product photos, ecommerce pages, social feeds, and news pages.
- Use short captions that can be localized without layout breakage.
- Make the DiffPixel panel and overlay result the primary visual focus.
- Do not rely on tiny UI text for comprehension.
